# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Install all dependencies (root)
npm install --legacy-peer-deps

# Run dev servers (server on :3001, client on :5173)
npm run dev

# Run all tests
npm test

# Server tests only (from root or server/)
cd server && npx vitest run
cd server && npx vitest run -- routes/auth     # single file

# Client tests only
cd client && npx vitest run

# TypeScript type-check (no emit)
cd server && npx tsc --noEmit
cd client && npx tsc --noEmit

# Generate DB migrations after schema changes
cd server && npx drizzle-kit generate

# Rebuild native better-sqlite3 bindings (after Node version change)
npm rebuild better-sqlite3
```

## Architecture

npm workspaces monorepo: `server/` · `client/` · `shared/`

### Shared types (`shared/types/index.ts`)
TypeScript interfaces used by both server and client: `Component`, `Product`, `ProductDetail`, `CategoryBreakdownItem`, `PublicProduct`, `AuthResponse`, etc. Import as `@jewelry/shared`.

### Server (`server/src/`)

**Entry point:** `index.ts` — starts Express, runs Drizzle migrations on boot, mounts all routers.

**Route factory pattern:** Every route file exports `default function createXxxRouter(db)` that takes a Drizzle db instance and returns an Express Router. This enables injection of an in-memory test DB. `index.ts` calls each factory with the real `db` singleton.

**Database:** Drizzle ORM + better-sqlite3. Schema in `db/schema.ts`. Migrations auto-generated by `drizzle-kit` into `db/migrations/`. The `db/index.ts` singleton reads `DB_PATH` env var.

**Key business logic** (pure functions, no DB, easy to test):
- `lib/calculations.ts` — `calcUnitCost`, `calcProductTotal`, `calcRecommendedPrice`, `calcCategoryBreakdown`
- `lib/tokens.ts` — `generateShareToken` (nanoid)

**`unit_cost_snapshot` design decision:** When a component is added to a product, its current `unitCost` is snapshotted into `product_components.unit_cost_snapshot`. This means retroactive price changes (re-ordering a component at a different price) do NOT alter historical product costs.

**Auth:** JWT via `middleware/auth.ts` → `requireAuth` middleware → injects `req.userId`. JWT secret from `JWT_SECRET` env var.

**File uploads:** Multer writes to `UPLOADS_DIR` env var (`./data/uploads` default). Filenames are random hex. Express serves `/uploads/*` statically.

### Client (`client/src/`)

**Routing:** React Router v6. Bottom tab nav: `/components` · `/products` · `/settings`. Auth-gated via `<PrivateRoute>` in `App.tsx`. Public share page at `/share/:token` (no auth).

**State:** Zustand (`store/authStore.ts`) persists JWT + user to localStorage. TanStack Query handles server state with `queryKey` patterns like `["components", { search, categoryFilter }]`.

**API layer:** `api/` files call Express via axios (`lib/axios.ts`). Axios interceptors inject Bearer token and handle 401 → logout.

**i18n:** All UI strings go through `react-i18next`. Translations in `locales/uk.ts`. Wire up new languages by adding a locale file and registering in `lib/i18n.ts`.

**PDF:** `@react-pdf/renderer` renders client-side. `lib/ProductPDF.tsx` is the PDF document component. Called from `ProductDetail.tsx` via `pdf(<ProductPDF .../>).toBlob()`. No server-side PDF generation.

**Mobile-first patterns:**
- `AppShell` + `BottomNav` for the shell layout. Bottom nav uses `pb-safe` for safe-area insets.
- `Modal` slides up from bottom (sheet pattern).
- `PhotoUpload` uses `capture="environment"` for direct camera access on mobile.
- All interactive targets have `min-h-[44px]` via `tap-target` utility class.

### Testing

**Server integration tests** (`server/src/__tests__/`) use `createTestDb()` → in-memory SQLite + migrations, then `createTestApp(db)` to build a test Express app. Each test file gets a fresh DB. Tests use Vitest + Supertest.

**Client tests** (`client/src/__tests__/`) use Vitest + React Testing Library + jsdom.

### Deployment (Railway)

- Single Node.js service serves both `/api/*` (REST) and `/*` (static React build from `client/dist`).
- SQLite file + uploads directory must be on a Railway persistent volume.
- Required env vars: `PORT`, `DB_PATH`, `UPLOADS_DIR`, `JWT_SECRET`, `CLIENT_URL`.
- Config in `railway.json`. Build: `npm install && npm run build`. Start: `node server/dist/index.js`.
